Симуляция Солнечной системы
Интерактивная 2D-симуляция движения планет Солнечной системы, написанная на Python с использованием ```tkinter``` и ```Pillow```. Проект позволяет наблюдать за орбитальным движением планет вокруг Солнца, регулировать скорость времени и масштаб, а также получать подробную информацию о каждом объекте по клику.

Возможности
Реалистичные орбиты: планеты движутся с разными скоростями в соответствии с их периодами обращения.

• Спутник: Луна вращается вокруг Земли.

• Интерактив: клик по любому объекту (Солнце, планета, Луна) открывает окно с его характеристиками.

• Управление временем: ползунок скорости позволяет ускорять или замедлять анимацию.

• Масштабирование: изменение масштаба для просмотра дальних планет.

• Орбиты: отображаются пунктирными линиями для наглядности.

• Изображения: при наличии PNG-файлов в папке ```images``` они используются вместо цветных кругов.

Требования
Python 3.6 или выше

Библиотеки:

• ```tkinter``` (обычно входит в стандартную поставку Python)

• ```Pillow``` (для работы с изображениями)

• ```math```, ```os``` (встроенные)

Установка и запуск
Клонируйте репозиторий:

```bash
git clone https://github.com/yourusername/solar-system-simulation.git
cd solar-system-simulation
```
Установите зависимости:

```bash
pip install Pillow
```
Поместите изображения планет в папку images (необязательно).
Ожидаемые имена файлов:
sun.png, mercury.png, venus.png, earth.png, mars.png, jupiter.png, saturn.png, uranus.png, neptune.png, pluto.png, moon.png.
Если файлы отсутствуют, объекты будут нарисованы цветными кругами.

Запустите программу:

bash
python sgshsh.py
(или как назван ваш основной файл)

Управление
▶ – запустить анимацию

⏸ – поставить на паузу

↺ – сбросить углы планет в начальное положение

 • Скорость – ползунок для изменения темпа времени (0–200% условной скорости)

• Масштаб – изменение масштаба от 0.2 до 3.0

• Сброс вида – вернуть масштаб и скорость к стандартным значениям

• Клик по объекту – показать подробную информацию в отдельном окне

Структура проекта • ```sgshsh.py``` – основной файл программы

• ```images/``` – папка для изображений планет (создаётся при необходимости)

Классы и архитектура
SolarObject – базовый класс для всех объектов (хранит координаты, размер, изображение).

• Planet – наследник SolarObject, добавляет параметры орбиты и движение вокруг Солнца.

• Moon – спутник, вращается вокруг планеты-родителя.

• Sun – статичный центральный объект.

SolarSystemApp – главный класс приложения, управляет интерфейсом и анимацией.

Примечания
Расстояния и размеры не соблюдают реальный масштаб (иначе планеты были бы не видны). Для наглядности используется коэффициент SCALE.

 • Периоды обращения приближены к реальным (в днях).

 • Проект создан в учебных целях и может быть доработан (добавление колец Сатурна, больше спутников, звёздного фона и т.д.).
